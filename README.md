## 我在做什么

我正在围绕 RAG Agent 构建一条更完整的智能应用链路：

```text
知识接入 → 数据清洗 → 索引构建 → 混合检索 → 结果重排
        → 上下文构建 → Agent 推理 → 工具调用 → 评估迭代
```

I build AI agents that turn language models into reliable, useful systems.
相比让模型单纯依赖参数知识，我更关注如何让 Agent：

`reasoning` / `tool use` / `memory` / `orchestration` / `evaluation`
- 从可信知识源中找到真正相关的信息
- 根据问题动态决定是否检索、检索什么以及检索多少
- 在有限上下文中组织高质量证据
- 结合记忆和工具完成多步骤任务
- 为最终回答提供可追踪、可验证的依据
- 在真实业务环境中保持稳定、可控和可评估

## 我的开发理念

1. **检索质量优先**：高质量上下文比盲目增加 Prompt 长度更重要。
2. **证据能够追溯**：Agent 的关键结论应该尽可能有来源、有依据。
3. **过程可以观察**：检索、推理和工具调用需要能够调试与评估。
4. **行动保持可控**：高风险操作必须设置权限、确认和安全边界。
5. **面向真实任务**：不仅追求 Demo 效果，也关注稳定性、成本和维护性。
